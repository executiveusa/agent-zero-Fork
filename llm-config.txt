SECTION: ANTI-PATTERNS (DO NOT DO)
================================================================================

DESIGN_ANTIPATTERNS:
    ✗ Bright neon colors (accessibility, UX)
    ✗ Auto-playing videos (UX nightmare)
    ✗ Low contrast text (WCAG violation)
    ✗ Hover-only critical info (mobile fails)
    ✗ Moving backgrounds (seizure risk, accessibility)
    ✗ Infinite scroll (pagination better for UX)
    ✗ Blinking elements (seizure risk)
    ✗ Multiple fonts (5+) (visual chaos)
    ✗ Comic Sans anywhere (ever)
    ✗ AI purple/pink gradients (clichéd)

DEVELOPMENT_ANTIPATTERNS:
    ✗ `console.log()` in production (pollutes logs)
    ✗ `setTimeout()` for delays (use proper async)
    ✗ Global variables (scope pollution)
    ✗ Callback hell (use async/await)
    ✗ Hard-coded URLs (use env vars)
    ✗ Unhandled promises (silent failures)
    ✗ Inline styles (CSS > inline)
    ✗ String concatenation for HTML (use templates)
    ✗ Synchronous I/O (always async)
    ✗ Comments that lie (outdated docs)

OPERATIONAL_ANTIPATTERNS:
    ✗ `docker run` without health checks
    ✗ Volumes without backups
    ✗ Logs inside containers (use stdout)
    ✗ Root user in containers (security)
    ✗ Secrets in Dockerfile (exposed in images)
    ✗ No monitoring (flying blind)
    ✗ Manual deployments (error-prone)
    ✗ No rollback strategy (stuck if broken)
    ✗ Ignoring warnings (debt accumulates)
    ✗ No staging environment (prod = test)

SECTION: TASK ROUTING ALGORITHM
================================================================================

WHEN_TASK_ARRIVES:
    1. Parse objective & estimated duration
    2. Classify by type (design, dev, ops, etc)
    3. Apply routing decision tree:
        
        IF duration < 2 minutes:
            agent = Switchblade  # Fast execution
        ELIF task_type == "design":
            agent = Cynthia      # Design specialist
        ELIF duration < 30 minutes:
            IF task_type == "ops" OR "devops":
                agent = Switchblade
            ELSE:
                agent = Cynthia or Switchblade
        ELSE:  # duration >= 30 minutes
            agent = ClaudeCode   # Long-running
        
        IF priority == "critical":
            bump_concurrency()   # Allow priority interrupt
    
    4. Save to /memory/tasks/current-task.json
    5. Notify agent (health check will pick it up)
    6. Monitor progress & update status

CONCURRENCY_LIMITS:
    ├── ClaudeCode: 1 (sequential only)
    ├── Cynthia: 3 (max concurrent design tasks)
    ├── Switchblade: 5 (max concurrent ops)
    └── Orchestrator: N/A (always running)

ERROR_RECOVERY:
    ├── Agent timeout? → Retry after 5 minutes
    ├── Container crash? → Auto-restart (restart policy: unless-stopped)
    ├── Memory corruption? → Rebuild from /memory backup
    ├── Health check fail? → Mark unhealthy, route tasks elsewhere
    └── Multiple failures? → Alert admin, pause task queue
